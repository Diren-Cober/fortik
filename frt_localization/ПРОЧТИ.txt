
    0. О ЛОКАЛИЗАЦИЯХ
Место файлов локализации ("loc_*.py") - в папке "frt_localization/localizations".
Каждый из них должен содержать пять питоновских словарей:
>	errs,
>   refs,
>	dbgs,
>	tags,
>	words.
Ключи словарей предопределены (за примером можете обратиться в имеющиеся в каталоге файлы "loc_*.py").
Замечание: имя локализации по умолчанию хранит свойство "dfl_loc" класса "frt_localization.locale.Locale".


    1. КАК СОЗДАТЬ СВОЮ ЛОКАЛИЗАЦИЮ
Для создания своей локализации сделайте следующее...
1. Создайте файл "frt_localization/localizations/loc_{$НАЗВАНИЕ_ЛОКАЛИЗАЦИИ}.py" (например, ".../loc_mine.py").
2. Убедитесь, что файл включает вышеуказанные словари с необходимыми ключами, и впишите свои (строковые) значения.
3. Найдите кортеж "loc_names" в прострастве класса Locale в файле "frt_localization/locale.py", он может выглядеть так:
.. ('rus', 'eng'); внесите '$НАЗВАНИЕ_ЛОКАЛИЗАЦИИ' в этот кортеж (он может принять следующий вид: ('rus', 'eng', 'mine')).

Теперь, добавив "--localization {$НАЗВАНИЕ_ЛОКАЛИЗАЦИИ}" в аргументы командной строки, вы обяжете фортик загрузиться
.. с использованием указанной локализации.
Помимо этого, фортик теперь будет проверять наличие файла "frt_localization/localizations/loc_{$НАЗВАНИЕ_ЛОКАЛИЗАЦИИ}.py"
.. при каждом последующем запуске.


    2. КАК УДАЛИТЬ ЛОКАЛИЗАЦИЮ
Для удаления локализации вам потребуется не только удалить её файл, но также и убрать её имя из кортежа "loc_names".


    3. КАК ОТКЛЮЧИТЬ ЛОКАЛИЗАЦИЮ
Для отключения локализации уберите её имя из кортежа "loc_names".
Если локализация отключена, фортик не знает о ней ничего, следовательно, не проверяет наличие её файла и
.. не может загрузиться с её использованием.


    4. КАК ПОДКЛЮЧИТЬ ИМЕЮЩУЮСЯ ЛОКАЛИЗАЦИЮ ИЛИ ДОБАВИТЬ СУЩЕСТВУЮЩУЮ
Для подключения имеющейся локализации просто добавьте её имя в кортеж "loc_names".
Добавляя же существующую, вы, помимо этого, должны убедиться, что её файл лежит в уже упомянутой папке
.. "frt_localization/localizations".
